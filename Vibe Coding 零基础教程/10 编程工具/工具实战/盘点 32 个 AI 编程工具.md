## 主流 AI 编程工具

#### Cursor

开局先来个王炸，Cursor 是目前全球最火的 AI 编程工具，没有之一。背后的公司估值已经接近 500 亿美元，堪称 AI 编程赛道的超级独角兽。

我是 Cursor 的重度用户，不止一个 Ultra 账号，每月消耗上万元，现在 80% 的项目代码、以及日常的工作流都用它来实现。

说下我看重 Cursor 的原因，首先 Agent 模式又快又稳，Claude、GPT、Gemini 各种主流模型都能一键切换，想用哪个用哪个，而且能够支持 100M 上下文。

其次开发体验很好，插件管理、MCP 配置、模型设置这些都有可视化界面，可以随时查看 Agent 运行状态和终端信息。

AI 编程能力加上可视化的开发体验，综合来看在同类工具里 **基本没有对手**。

![](https://pic.yupi.icu/1/1_cursor.png)

而且 Cursor 的更新迭代非常频繁，到现在已经有 Cloud Agents 云端并行、Browser Use 浏览器自动化测试、Design Mode 直接在界面上选中元素让 AI 改代码、内置 AI 生图等等能力了。还提供了命令行版本和网页版，手机上打开网页就能随时随地操作 Agent 干活。

不过跟大多数国外 AI 编程工具一样，国内使用有一定的网络门槛。免费版只能使用 Auto 模式提供的免费额度，即使自己配置了国内模型也不能用。

此外，Cursor 是基于 VS Code 开源版本实现的，但界面整体不如原生 VS Code 精致，Markdown 的阅读体验能优化下就更好了。

看一下价格，真心不便宜啊！Ultra 一个月 200 刀，账单烧的嘎嘎猛，这可能也是很多人弃用 Cursor 的原因。

| 套餐 | 价格 |
|:-----|:-----|
| Free | 免费（有限 Agent 和补全额度） |
| Pro | $20/月 |
| Pro+ | $60/月 |
| Ultra | $200/月 |
| Teams | $40/用户/月 |

还有朋友诟病 Cursor 是觉得它们没有强大的自研模型，觉得未来可能会因为模型被卡脖子，拼不过 Claude Code 和 Codex。

不管怎么说，这个工具确实彻底改变了我的工作方式，而且几乎没有解决不了的问题。如果有，那就是 Tokens 没烧够。



#### GitHub Copilot

接下来是曾经的王者 GitHub Copilot。

Copilot 的产品形态非常丰富，有 VS Code 插件、GitHub 网页版、还有命令行终端，覆盖面是所有工具里最广的。毕竟是微软官方出品，和 VS Code 的集成是真的好，界面优雅、交互体验我觉得是所有工具里最舒服的。MCP 和 Skills 等插件扩展也可以可视化管理，能力也很齐全。

![](https://pic.yupi.icu/1/2_copilot.png)

我还带大家用 GitHub Copilot 网页版搭建过一个能自主开发、上线、修 Bug 的智能体。

![](https://pic.yupi.icu/1/1774937381654-bde29eb7-3d0f-4869-a065-f7ddb09950dd-20260514180312605.png)

曾经 Copilot 的性价比巨高，每月 10 刀就能用上 Opus 模型，还有 30 天免费试用，那会儿真的是王者。但现在官方扛不住成本了，好日子到头了，使用门槛变高了不少。6 月 1 号开始还要切换到按 Token 计费的模式，上下文也有 20 万的限制。

跟其他国外工具一样，国内使用也有一定的网络门槛。

| 套餐 | 价格 |
|:-----|:-----|
| Free | 免费（50 次 Agent + 2000 次补全/月） |
| Pro | $10/月 |
| Pro+ | $39/月 |
| Business | $19/用户/月 |

免费版的额度对于轻度使用来说还算够用，Pro 版 $10/月也不贵。但要注意 6 月起计费模式要变了。



#### Codex

OpenAI 的 Codex 最近更新速度嘎嘎快，奥特曼说它正在经历 ChatGPT 时刻。

Codex 的界面主打一个简洁，第一眼看上去甚至不像 AI 编程工具，更像个 AI 聊天助手。但实际上功能很全面，MCP 扩展、Skills 技能包、插件市场、自动化、Git 集成、Browser Use、Computer Use 这些工程能力基本都具备了。最近甚至还搞了个 AI 桌面宠物加情绪价值。。。

![](https://pic.yupi.icu/1/3_codex.png)

Codex 也独创了一些 AI 编程玩法，比如推出了 `/goal` 命令，可以设定一个持久目标让 AI 自主工作数小时，不需要人工干预。

不过 Codex 目前 **默认** 只能使用 GPT 系列模型（可以用 CC Switch 等工具快速修改），编程能力上 GPT-5.5 整体还是不错的，基本能一把梭搞定全栈项目，但前端表现中规中矩，整体不如 Claude Opus 和使用了 Claude 模型的 Cursor。而且对我来说目前 GPT-5.5 的 20 多万的输入上下文真心不够用，这是致命伤！

Codex 的易用性也差一些，比如配置 MCP 服务要手写信息，不像 Copilot 可以一键安装，查看代码的体验也不如基于 VS Code 生态的 Cursor 方便。

跟其他国外工具一样，国内使用需要网络工具，也需要 ChatGPT 订阅。

| 套餐     | 价格          |
| :------- | :------------ |
| Plus     | $20/月        |
| Pro      | $200/月       |
| Business | 按 Token 计费 |

我自己是 Plus 会员，做一个全栈项目大概用掉 5 小时额度的一半，1 周差不多能做 10 个完整小项目。

等什么时候能让用户轻松自由切换模型了、上下文达到 100 万了、或者优化一下读写代码的体验，Codex 还有很大的提升空间。

再次强调，我并没有说 Codex 不好，大家不要激动。我写过多篇 Codex 实战教程，所有的理由也都列举出来了，我只是希望他能做的更好。



#### Windsurf

我对 Windsurf 最深的印象是，之前经历了 OpenAI、Google 的收购风波。

巨头能看上，说明产品本身有东西。

Windsurf 核心的 Agent 叫 Cascade，能跨多个文件自主规划和执行多步编辑。被收购之后还集成了 Devin，可以一键把复杂任务委派给云端执行，本地和云端双路协作。它还有个 Memories 功能，Agent 会逐渐学习你的代码库架构和编码风格，用得越久建议越精准。

![](https://pic.yupi.icu/1/4_windsurf.png)

Windsurf 的价格跟 Cursor 基本持平。不过跟 Cursor 比起来，模型灵活度和社区生态还是有差距的，国内同样有网络门槛。

| 套餐 | 价格 |
|:-----|:-----|
| Free | 免费 |
| Pro | $20/月 |
| Max | $200/月 |
| Teams | $40/用户/月 |



#### Trae 国际版

Trae 是字节跳动的 AI IDE，分为国内版和国际版，二者在模型、定价、多模态能力等地方的差异都很大，所以我把它们分开讲。

国际版默认集成了 GPT、Gemini、DeepSeek 等多家主流模型，支持用自己的 API Key 使用 Claude。功能上主打的是 SOLO 模式，具备长期记忆，能全自动开发、测试、调试和部署项目。而且新上线了移动端 App，手机上也能操作。

![](https://pic.yupi.icu/1/5_trae.png)

不过国际版和其他国外工具一样有网络门槛。

看看它的定价，Pro 版只要 $10/月，差不多是 Cursor 的一半价格，性价比不错。

| 套餐 | 价格 |
|:-----|:-----|
| Free | 免费（基础额度） |
| Lite | $3/月 |
| Pro | $10/月 |
| Pro+ | $30/月 |
| Ultra | $100/月 |



#### Kiro

Kiro 是 AWS 推出的 AI IDE，跟其他工具最大的不同是它主打「规格驱动开发」。

简单说就是你不能一上来就让 AI 写代码。Kiro 会要求你先用自然语言描述需求，然后它自动生成需求文档、设计文档和任务清单这 3 份文件，你审批通过之后它才会开始写代码。

有点像计划模式，但是比计划模式更严谨。所有设计意图都版本控制在仓库里，还有 Agent Hooks 功能，比如文件保存的时候能自动触发测试和文档更新。

![](https://pic.yupi.icu/1/6_kiro.png)

但对于绝大多数刚上手 AI 编程的同学来说，改个小功能还得先走一遍规格审批，把简单的事搞麻烦了。

价格方面跟 Cursor 相当，Kiro 的默认模型在国内是可以直接使用的，有一定的免费额度。

| 套餐 | 价格 |
|:-----|:-----|
| Free | 免费（50 Credits） |
| Pro | $20/月 |
| Pro+ | $40/月 |
| Power | $200/月 |



#### Zed

Zed 是轻量级的 AI 编辑器，刚发布 1.0 正式版。

它主打的就是极致性能，根据官方数据，冷启动只要 180 毫秒，内存占用才 100 多 MB。

它还内置了 120fps 的实时多人协同编辑，不需要装任何插件就能多人同时写代码，AI 功能方面也有 Agent 面板和并行多 Agent。

![](https://pic.yupi.icu/1/7_zed.png)

不过整体成熟度跟 Cursor 还是有差距的，扩展生态也比较小。说实话，现在用 AI 编程，大部分时间都在等 AI 思考和输出，编辑器本身快一点儿的优势感知不太明显。

Zed 的 Pro 版定价 $10/月，差不多是 Cursor 的一半价格，学生还可以免费用一年，性价比还行。

| 套餐 | 价格 |
|:-----|:-----|
| Personal | 免费 |
| Pro | $10/月 |
| Business | $30/seat/月 |



#### Antigravity

Antigravity 是 Google 推出的 Agent-first IDE，它集成了编辑器、终端和内置浏览器三个工作面，Agent 可以横跨这三个面自主执行任务，最多支持 5 个 Agent 并行。你只需要审批结果就行，Agent 还会生成截图、录屏、代码 diff 这些可验证的交付物，方便你确认质量。模型方面不仅支持 Google 自家的 Gemini，Claude Opus 也能用。

![](https://pic.yupi.icu/1/8_antigravity.png)

不过由于是 Google 的产品，国内使用嘛，你懂的。。

看一下价格，Pro 版跟 Cursor 差不多，Ultra 版 $250/月 比 Cursor 的 Ultra 还贵一些。

| 套餐 | 价格 |
|:-----|:-----|
| Free | 免费预览 |
| AI Pro | $19.99/月 |
| AI Ultra | $250/月 |

Antigravity 自从发布之后，更新节奏不如 Cursor 和 Codex 那么频繁，最近存在感有点低了。



#### Trae 国内版 / Qoder / 腾讯 CodeBuddy

字节的 Trae 国内版、阿里的 Qoder、腾讯的 CodeBuddy，是国内 AI 编程 IDE 的御三家。功能比较接近，最大的共同优势就是 **国内直连无障碍，不需要任何网络工具**。

**Trae 国内版** 跟国际版有些差距，模型换成了豆包 Seed Code、DeepSeek、Kimi 等国产模型，但核心 AI 编程功能完全免费无限制（就是得排队）。

![](https://pic.yupi.icu/1/9_trae_cn.png)

**Qoder** 底层用的自研 Qwen-Coder 模型，国内版也支持接入第三方模型。它有个 Repo Wiki 功能，可以自动生成项目知识图谱，适合理解大型代码库。

![](https://pic.yupi.icu/1/9_qoder.png)

**腾讯 CodeBuddy** 在产品形态上比较全面，有独立 IDE、VS Code/JetBrains 插件、还有命令行版本。CodeBuddy 最大的亮点是跟微信生态的深度集成，支持微信小程序开发和企业微信 Bot 集成，搞微信相关开发用它比较合适。

![](https://pic.yupi.icu/1/9_codebuddy.png)

三个工具的价格：

| 工具 | 免费版 | 付费版 |
|:-----|:-------|:-------|
| Trae 国内版 | 核心功能免费无限制 | 企业版 69 元/人/月起 |
| Qoder | Community 免费 | Pro $20/月起，Teams 中国区 300 元/席位/月 |
| CodeBuddy | 个人免费（限时） | 企业旗舰 198 元/人/月 |

这三家都还在快速迭代的阶段，功能上跟 Cursor 这种国际顶流比还有差距，但好在国内直连、免费、中文体验好，对于初学者和预算有限的同学来说是非常好的入门选择。

我真心希望国内的 AI 编程工具能越做越好，尽早追上国际一线水平。



#### OpenCode

OpenCode 是开源的终端 AI 编程工具，GitHub 上有十几万 Star！

OpenCode 的优势在于完全免费开源，支持非常多的模型提供商，提供了 Scout Agent 代码库研究功能，对代码库的理解和导航比较方便。而且还出了桌面应用和 VS Code 扩展。

![](https://pic.yupi.icu/1/10_opencode.png)

我在它刚火的时候就试过了，当时印象最深的是前端界面做的非常漂亮，用户体验上碾压 Claude Code。但后端工程能力跟 Claude Code 还是有差距的。

那问题来了，如果只是追求前端使用方便，我为啥不直接用 Cursor 呢？如果追求极致的编程能力，Claude Code 又更强。

不过 OpenCode 提供了一些免费模型，能够帮小白练手。



#### Warp

Warp 是定位比较特殊的 AI 终端工具，最近刚刚开源了。

它跟 Claude Code、OpenCode 这类终端 AI 编程工具有个本质区别，其他工具是你在终端里运行一个 AI 程序，而 Warp 本身就是一个终端软件，给整个终端增加了智能化，像是终端和桌面端的混合体。

![](https://pic.yupi.icu/1/11_warp.png)

不过实际体验下来，我觉得有点儿不上不下。既然要用 AI 编程，我不如直接去用 Cursor 这种有完整界面的工具，效率更高。而且 Warp 缺少新用户的引导，小白上手会有点儿懵。

看一下价格，也不便宜。

| 套餐 | 价格 |
|:-----|:-----|
| Free | 免费（前 2 月 150 Credits，之后 60 Credits/月） |
| Build | $18/月起 |
| Max | $180/月起 |
| Business | $45/用户/月 |

最气人的是，我还没用几次就给我封号了？还是说想让我付费？简直莫名其妙。



#### Goose

Goose 是一个开源 AI Agent，目前已经捐给了 Linux 基金会。

它有个 Recipes 功能，可以把扩展、提示词和设置打包成可复用的工作流模板分享出去，MCP 扩展生态也很丰富，但没什么其他的亮点了。

![](https://pic.yupi.icu/1/12_goose.png)

不过 Goose 的知名度太低了，整体生态过于小众。



#### Aider

Aider 是开源的终端 AI 编程工具，GitHub 上有好几万 Star。

它的特点是跟 Git 深度绑定，改完代码自动 commit 还顺带写好描述。还有个 Architect 模式，可以用强模型规划任务、便宜模型来写代码，节省 API 费用。

但是我试了一下，使用体验一言难尽，连调用工具都能失败，给我一种 23 年 AI 刚出时的感觉，怪不得生态那么小众。。

![](https://pic.yupi.icu/1/13_aider.png)



#### Claude Code

如果说 Cursor 是可视化 AI 编程的王者，那 Claude Code 就是命令行 AI 编程的绝对霸主！

Claude Code 是 Anthropic 推出的终端编程 Agent，处理复杂任务的能力没得说，妥妥的 Number 1。在 SWE-bench 基准测试中拿下了 80.8% 的行业最高分。

它还是很多 AI 编程新能力和规范的引领者。比如 CLAUDE.md 持久记忆文件，最早就是在 Claude Code 中推出的，后来被行业标准化为 AGENTS.md，其他工具纷纷跟进。还有 Hooks 事件驱动系统、Subagents 子智能体架构、Agent Teams 多 Agent 协作这些机制，也都是通过 Claude Code 率先落地的。

![](https://pic.yupi.icu/1/14_claudecode.png)

对了，前段时间 Claude Code 不是还把源码送给大家学习了？这是推动整个行业进步啊！

不过它的使用是有一定门槛的。一方面需要有命令行基础，对于大多数只会可视化操作的同学来说不太友好。随便举个例子，Claude Code 怎么查看终端任务的进行状态？你让小白摸索试试？

另一方面，Claude Code 默认使用自家的 Claude 模型，国内不能直接使用。不过现在可以用 CC Switch 之类的工具切换到国内模型来解决。

虽然交互体验上不如 Copilot、Cursor、Codex 这些可视化工具直观，但命令行有自己的优势，灵活方便、在任何终端环境里都能用。

如果你想追求最强的 AI 编程能力，还是建议学习一下命令行操作。我写过很多篇 Claude Code 的实战教程，大家都能跟着学。

再看一下价格。每月 20 刀的 Pro 计划主要提供 Sonnet 模型，想在 Claude Code 中用最强的 Opus 模型得开每月 100 刀的 Max 套餐，还是挺贵的。

| 套餐 | 价格 |
|:-----|:-----|
| Pro | $20/月（Sonnet 4.6） |
| Max 5x | $100/月（Opus 4.6） |
| Max 20x | $200/月 |
| API | 按量计费 |

Claude Code 作为命令行 AI 编程的绝对标杆，AI 编程能力最强、新范式的引领者，非常值得关注和学习。

主流的 AI 编程工具就介绍完了。除了这些之外，还有几类产品也能拿来做 AI 编程，虽然定位不完全一样，但各有各的用处。



## AI 编程插件

不知道你们现在还用不用「AI 编程插件」？就是安装在 IDE 里，可以用来代码补全什么的。

我是几乎不用了。。。

这些插件非常多，而且功能高度重叠，所以我讲的时候会偷个懒儿，合并一下同类项。



#### 通义灵码 / 百度 Comate / CodeGeeX

先说国内 AI 编程插件的早期三幻神，阿里的通义灵码、百度的 Comate 和智谱的 CodeGeeX。

这三家的核心能力都差不多，代码补全、代码解释、单元测试生成、多语言支持这些。国内可以直连使用，中文体验很好。

![](https://pic.yupi.icu/1/%E9%80%9A%E4%B9%89%E7%81%B5%E7%A0%81.png)

不过 25 年之后，这几个产品就没有太大的风浪了，在 Cursor、Claude Code 这些工具面前存在感越来越弱。



#### Cline

Cline 是开源 AI Agent 插件里的标杆，GitHub 上有 6 万多 Star。

相比前面那三家国产插件，Cline 最大的优势在于 Agent 能力更强，支持 Plan & Act 双模式和 Checkpoints 回滚机制，改坏了可以一键回到之前的状态。

![](https://pic.yupi.icu/1/cline.png)

Cline 在 VS Code 插件市场的安装量有几百万，在全球开发者中口碑不错，社区活跃度也一直在线。



#### Kilo Code / JetBrains Junie / Continue / Augment Code

然后是一些整体存在感不算高的 AI 编程插件，我估计很多同学都没听说过，我放在一起讲。

**JetBrains Junie** 是 JetBrains 自家的 AI Agent，它可以完全委派任务自主执行，利用 IDE 的语法树做编译器级别的纠错。但是只支持 JetBrains 系 IDE，不支持 VS Code，而且需要改账号地区才能用。

![](https://pic.yupi.icu/1/junie.png)

**Kilo Code** 本质上是 Cline 的一个 fork 分支，走了商业化路线。支持的模型和平台很多，覆盖 VS Code、JetBrains、CLI、移动端等，还有 Memory Bank 持久化项目上下文的功能。

**Continue** 模型自由度很高，不过配置确实比较复杂，上手门槛偏高。

**Augment Code** 号称大型代码库的专家，Context Engine 可以索引大量文件。但已经开始封区了，国内注册困难，价格也不便宜。

这些插件虽然各有特色，但在当前 AI 编程插件赛道里整体存在感偏弱，功能上也很难跟主流 AI IDE 竞争。



#### Tabnine

我记得 Tabnine 应该是我上大学那会儿很火的代码补全插件，但现在已经全面转向企业市场了。免费版停了，个人付费版也在下架，目前只剩企业套餐，普通开发者基本用不了。

![](https://pic.yupi.icu/1/tabnine.png)



## AI 低代码平台

接下来是 AI 低代码平台，这些平台基本都长一个样儿。简单来说就是你输入一句提示词，然后它帮你生成完整的应用，还能自动部署上线。

这类平台最重要的就是用起来方便，前端要美、后端要稳，生成速度要快。

为了公平对比，我全部使用免费版账号，用同一个提示词测试了所有平台：帮我完全仿照鱼皮 AI 导航网站，开发一个美食导航网站。

这个任务需要联网搜索我的网站，既考察 Agent 能力，又考察前端还原能力。

下面我们依次看看各个 AI 低代码平台的表现。



#### Manus

Manus 耗时最长，花了 15 分钟左右，但 **还原度是最高的**，把我网站的风格抓得很准。

![](https://pic.yupi.icu/1/manus.png)

我之前对 Manus 的印象其实是不好的，因为有一次我做视频给大家演示 Manus 结果被骂的很惨。。。

大家都觉得 Manus 的故事比它的产品还精彩，爆火、融资、裁员、跑路新加坡、被 Meta 20 多亿收购、被国家发改委叫停，堪称一部大型连续剧，AI 都不敢这么演。

不过没想到，我这次测试下来 Manus 表现竟然是最好的。Manus 主要擅长的是前端生成和自动部署，后端能力相对有限，更适合做展示型应用和 MVP 原型。

产品价格 $20/月起步。



#### v0.dev / Lovable / Replit Agent

**v0.dev** 是 Vercel 出品的，就是做知名前端框架 Next.js 的那家公司。用时 3 分 57 秒完成，**也还原了我网站的样式**，效果非常好。

它最擅长的还是前端 UI 生成，虽然官方已经支持通过 Next.js Server Actions 和 API Routes 做一些后端逻辑了，但实际生成的后端代码质量还不太行，适合做原型不适合直接上生产。

![](https://pic.yupi.icu/1/v0dev.png)

**Lovable** 生成速度最快，3 分 08 秒就搞定了，但 **没有还原我网站的样式**。它更擅长从零开始生成 MVP，深度捆绑了 Supabase 做认证和数据库，还有 Stripe 支付脚手架，对非技术背景的同学比较友好。

![](https://pic.yupi.icu/1/lovable.png)

**Replit Agent** 花了 6 分钟，**也没有还原我网站的样式**。相比其他两个产品，也没有什么独特的功能。

![](https://pic.yupi.icu/1/replit.png)

价格上几个平台都差不多：

| 平台 | 免费版 | 付费版 |
|:-----|:-------|:-------|
| v0.dev | Free（$5 月度积分） | Premium $20/月 |
| Lovable | Free（30 积分/月） | Pro $25/月 |
| Replit Agent | Free 可用 | Core $20/月，Pro $100/月 |



#### Bolt.new

我自己免费的 AI 编程教程中，使用 Bolt.new 作为小白开发 AI 应用的入门 Demo 制作工具。

结果这次实测下来，翻车了。。。它把需求想复杂了，直接进入了 Plan 模式，结果把我免费账号的 Tokens 都烧完了，却一行代码都没写出来。

![](https://pic.yupi.icu/1/boltnew.png)

赣！你让新手怎么用啊？

整体来说，这类 AI 低代码平台更适合零基础的新手快速入门，如果你追求更灵活强大的 AI 编程能力，还是用最开始提到的主流 AI 编程工具吧。



## AI Agent

最后说一下 AI Agent 类的产品。

AI Agent 是一种能自主完成复杂任务的智能助手，你给它一个目标，它自己拆解任务、调用工具、一步步执行。

虽然下面这些并不是专业的 AI 编程工具，但它们也能拿来做 AI 编程，所以放到这里给大家涨涨姿势。

#### OpenClaw / Hermes Agent

OpenClaw 是一个开源的全能 AI 助手，它最大的优势是可以接入微信、钉钉、飞书等各种平台，随时随地给 AI 派活，做个 Demo 确实很方便。

![](https://pic.yupi.icu/1/openclaw.png)

我在它爆火的时候给大家出过很多篇教程，从保姆级安装、到高阶玩法、再到项目实战全都有。

![](https://pic.yupi.icu/1/OpenClaw%E6%95%99%E7%A8%8B%E6%BC%94%E7%A4%BA-%E6%8A%80%E8%83%BD%E7%B3%BB%E7%BB%9F.png)

Hermes Agent 也是现在最流行的 Agent 产品之一，跟 OpenClaw 属于同一类产品。但是它跟 OpenClaw 最大的区别在于它有「自进化能力」，每次解决问题后会自动生成可复用的技能，越用越聪明。

![](https://pic.yupi.icu/1/hermes.jpg)

我自己拿它们做过一些 Demo，只能说「能用」，但真要做正儿八经的项目，稳定性远不如专业 AI 编程工具。要做完整项目的话，大概率还是得用 Cursor 或者 Claude Code 来接管。而且 tokens 烧麻了！普通人要么不会玩，要么玩不起。



## 写在最后

总结一下。如果你刚入门 AI 编程，我推荐 **Trae 国内版** 零成本练手，进阶的话用 **Cursor** 做实战开发，想更深入就学习 **Claude Code** 的命令行操作。此外，Codex 也非常值得关注，等它能自由切换模型了，说不定会超越 Claude Code 呢？

整体来看，AI 编程能力 = 模型 + 工程。国产工具在工程层面的差距正在快速缩小，国外工具新出什么能力，国内很快就能跟上。但在底层模型的推理能力上还有差距，很期待国产 AI 能把价格打下来，让更多人用得起。

学会了这些内容，你就能根据自己的需求和预算，选择最适合自己的 AI 编程工具。

如果你想继续学习更多 AI 编程工具和实战技巧，可以阅读本教程编程工具板块的其他文章。
