# 我的 AI 工具箱推薦

> 我今年用得最多的 AI 產品 Top 20



你好，我是程式設計師魚皮，前騰訊全端開發，全網 200 萬粉的 [AI 程式設計博主](https://space.bilibili.com/12890453)，也是 [AI 導航](https://ai.codefather.cn) 和 [程式設計導航](https://www.codefather.cn) 等 10+ 自研產品的創造者。

在前面的文章中，我們系統學習了 AI 程式設計工具的各個類型：從 AI 模型選擇、零程式碼平台、AI 程式碼編輯器、命令列工具、IDE 插件、AI 智能體平台，到輔助工具集。

那問題來了：

- 這麼多工具，魚皮你自己到底用哪些？
- 實際工作中，這些工具怎麼搭配使用？
- 有沒有一些實戰經驗可以分享？

這篇文章，我會分享 2025 年我使用最多的 AI 產品 Top 20，以及我的實際使用心得，希望能給你一些參考。



## 一、AI 程式設計工具

作為一名程式設計師，AI 程式設計工具是我用得最多的。下面是我今年的主力程式設計工具。



### Cursor

[Cursor](https://cursor.com/) 是我心中開發體驗和 AI 工程能力 Top 1 的 AI 編輯器。做大專案主要靠它，無論是寫程式碼、改程式碼、還是重構程式碼，Cursor 的 Agent 模式都能幫我搞定。

我特別喜歡它的 Tab 補全功能，寫程式碼的時候 AI 會自動預測我要寫什麼，按 Tab 就能接受建議。這種感覺就像有個老司機坐在旁邊，你剛想到要寫什麼，他就已經幫你寫好了。

而且 Cursor 可以自由切換 Claude、GPT、Gemini 等多個模型，還能使用自己的 API Key，我可以根據不同任務選擇最合適的模型。

不過 Cursor 最大的問題就是價格死貴。我之前分享過帳單，一個多月就燒了 1 萬多塊。所以我現在會搭配一些免費工具使用，避免預算爆炸。

![](https://pic.yupi.icu/1/yupicursorbilling.png)



### Claude Code

[Claude Code](https://claude.ai/code) 是我用得最多的命令列 AI 程式設計工具。它的自動化程度非常高，不僅能生成程式碼，還能自動執行命令、安裝依賴、建立檔案。

我一般用它來做一些簡單的任務，或者快速搭建專案框架。比如我想做一個圖片壓縮工具，直接在命令列裡跟 AI 說一句話，它就能自動幫我搭好整個專案。

![](https://pic.yupi.icu/1/1764146174167-3018c8f3-0ad9-4a4f-9764-246f33b33203.png)

而且 Claude Code 支援 Skills，可以把一些常用的規範和流程寫成 Skill，讓 AI 自動遵循，對於避免重複工作和團隊協作特別有用。

因為 Claude Code 在國內有限制，我一般會配合國產 AI 大模型使用，比如智譜 GLM。這樣既能正常使用，成本也更低。



### Cline

[Cline](https://cline.bot/) 是一款 AI IDE 插件，是我的備胎之一。當 Cursor 的預算用完了，或者我想在 VS Code 裡開發時，就會用 Cline。

![](https://pic.yupi.icu/1/image-20260108222935455.png)

Cline 最大的優勢是完全免費，而且功能很全面，基本上能做到 Cursor 80% 的事情。雖然體驗沒有 Cursor 那麼流暢，但考慮到它是免費的，已經非常不錯了。

而且 Cline 支援多個平台，不僅是 VS Code、JetBrains 系列也能用。這意味著我在不同編輯器之間切換時，都能用同一個工具，保持一致的體驗。



## 二、AI 多媒體創作工具

除了程式設計，我還會用 AI 做內容創作。下面是我用得最多的多媒體創作工具。



### Nano Banana

Nano Banana 是今年最炸裂的 AI 生圖工具，我用它來生成漫畫知識講解圖片和文章封面圖。

它對中文的支援特別好，不像 Midjourney 那樣需要用英文提示詞。而且生成的圖片風格很統一，特別適合做教程配圖。

你看到的這種動漫風格的圖片，都是用 Nano Banana 做的：

![](https://pic.yupi.icu/1/toolstype%25E5%25A4%25A7.jpeg)

自從用了 Nano Banana，我就徹底和 Midjourney 說再見了。



### 即夢 AI

[即夢 AI](https://jimeng.jianying.com/) 是字節的一站式 AI 創作平台，支援文字生圖、圖生影片等功能。

我主要用它來生成影片空鏡畫面。比如我做教程影片時，需要一些背景畫面，就會用即夢 AI 生成。它的影片生成速度很快，而且質量不錯。

而且即夢 AI 是國產平台，訪問速度快，也不用擔心被牆的問題。



### Veo 3

[Veo 3](https://aistudio.google.com/models/veo-3) 是 Google 最先進的 AI 影片生成模型，我用它來做一些有創意的短影片。

Veo 3 的影片質量確實很高，物理規律、光影效果都很真實。但是價格死貴，所以我只在需要高質量影片的時候才會用它。

![](https://pic.yupi.icu/1/1751177650505-09aecbd1-a858-4a01-a74d-62120f074c98.png)



### Eleven Labs

[Eleven Labs](https://elevenlabs.io/) 是 AI 語音合成工具，模仿聲音效果特別逼真。

而且 Eleven Labs 支援多種語言和聲音風格，可以根據不同場景選擇不同的聲音，我會用它來合成一些創意影片的配音。

![](https://pic.yupi.icu/1/image-20260109100507368.png)



## 三、AI 大模型

AI 大模型是所有 AI 工具的基礎。下面是我今年用得最多的幾個大模型。



### 智譜 GLM

[智譜 GLM](https://bigmodel.cn/) 是我用得最多的國產大模型，程式設計能力很強，中文理解能力也很優秀。

我主要用它搭配 Claude Code 生成非商業級的全端專案。因為 GLM 的 API 價格很便宜，而且速度快，特別適合做一些小專案或者快速驗證想法。

如今智譜的發展也是嘎嘎猛，從 GLM-4.6 開始程式設計能力就有了很大提升，在一些測評中甚至超過了 Claude Sonnet！公司上市也是理所應當的。



### 阿里通義千問

通義千問是我們團隊產品接入最多的大模型。它的優勢是生成速度快、API 對接方便，還可以利用 [阿里雲百煉平台](https://bailian.console.aliyun.com/) 調試。

![](https://pic.yupi.icu/1/1752569401445-7023e2c1-f709-4c22-af86-63ee1872c1cd.png)

我們的產品用它來生成簡單的內容或者做意圖識別。比如用戶輸入一句話，我們用通義千問來判斷用戶想做什麼，然後調用相應的功能。

而且通義千問有免費額度，對於小專案來說夠用一陣子了。



### DeepSeek

[DeepSeek](https://www.deepseek.com/) 作為曾經是爆款，完全免費而且程式設計能力很強。不過最近無論是產品生態還是出新模型的速度都略遜一籌。

我現在主要用它來校驗專業技術知識。比如我寫文章時，不確定某個技術點是否正確，就會問問 DeepSeek。反正它是免費的，所以我可以隨便問，不用擔心成本。

![](https://pic.yupi.icu/1/image-20260109100658239.png)



### Gemini

[Gemini](https://gemini.google.com/) 具備強大的推理能力和多模態理解能力，而且支援超長上下文（100 萬 Token）。

我主要用它來分析大型專案。比如我接手一個幾百個檔案的專案，想快速了解專案結構，就會用 Gemini 來分析。

我還會利用 Gemini 網頁版來搜集一些國外的技術和產品調研，給出的報告很專業。

![](https://pic.yupi.icu/1/image-20260109100840547.png)

不過 Gemini 在國內有一定的使用門檻，你懂的。



## 四、AI 開發框架

除了直接使用 AI 工具，我還會用一些 AI 開發框架來構建 AI 應用。



### Spring AI

[Spring AI](https://spring.io/projects/spring-ai) 是 Java AI 開發的標配，提供統一 API 簡化大模型集成。

我們團隊做 Java 專案時，基本上都會用 Spring AI。它把各種大模型的 API 統一成一套接口，這樣開發者可以很方便地切換不同的模型，不用改程式碼。

![](https://pic.yupi.icu/1/image-20260109100909041.png)

而且 Spring AI 提供了很多高級功能，比如對話記憶、RAG、工具調用等，讓我們可以快速構建複雜的 AI 應用。



### LangChain4j

[LangChain4j](https://docs.langchain4j.dev/intro) 是 Java 版的 LangChain，支援聲明式語法，開發體驗巨爽，適合構建複雜 Agent。

我特別喜歡它的鏈式調用語法，寫起來很優雅。而且它的文件很完善，社區也很活躍，遇到問題可以到 GitHub Issues 區多看看，容易找到解決方案。

我帶 [程式設計導航](https://codefather.cn/) 的魚友們做對標大廠的 [AI 零程式碼生成平台專案](https://www.codefather.cn/course/1948291549923344386) 時，就是用的 LangChain4j + LangGraph4j，能夠很方便地構建複雜的 AI 工作流。

![](https://pic.yupi.icu/1/1753332332820-9ec614de-65a2-496d-b9b2-dc89c20d06c9.png)



### Vercel AI Gateway

[Vercel AI Gateway](https://vercel.com/ai-gateway) 是 AI 模型網關服務，讓你能統一調用數百個 AI 模型。

它提供了統一的 API，我不用去學習每個模型的 API 文件，直接用 Vercel AI Gateway 就行。

![](https://pic.yupi.icu/1/1760687990497-90720fbb-0df6-4ede-87b8-64b8702994e9-20251028181254840.png)

我用它來開發一些創意專案。比如我想做一個 AI 工具，讓用戶可以選擇不同的模型，Vercel AI Gateway 就能幫我很方便地實現這個功能。



### 阿里雲百煉

[阿里雲百煉](https://bailian.console.aliyun.com/) 是企業級 AI 應用開發平台，支援 RAG 知識庫、流程編排等功能。

我們團隊的一些知識庫就用它來管理。比如我們有很多技術文件，用百煉的 RAG 功能可以很方便地讓 AI 基於這些文件回答問題。

![](https://pic.yupi.icu/1/1744177887403-6373c63c-d410-49ba-a71c-556389536376.png)

而且百煉提供了可視化的流程編排功能，不用寫程式碼就能構建複雜的 AI 工作流。

![](https://pic.yupi.icu/1/1745388167957-d7995436-fc2f-410e-bc59-4f6db6544359.png)



### OpenRouter

[OpenRouter](https://openrouter.ai/) 是統一的 AI 模型路由平台。它支援幾百個模型，而且可以很方便地切換，不用去每個平台註冊帳號。

每次有新模型發布，我都會在 OpenRouter 上試試，用它來體驗最新的大模型能力。

![](https://pic.yupi.icu/1/image-20260109101446266.png)

對於想要嘗鮮的開發者來說，OpenRouter 是個很好的選擇。



## 五、AI 瀏覽器插件

AI 瀏覽器插件讓我在瀏覽器裡隨時使用 AI，特別方便。

### Monica

[Monica](https://monica.im/) 是瀏覽器裡的全能 AI 助手，網頁總結、影片總結、翻譯、寫作、圖片處理一應俱全。

![](https://pic.yupi.icu/1/image-20260109101602426.png)

我每天都在用 Monica。比如看長文章時，用它總結一下核心內容；看英文文件時，用它翻譯；看程式碼倉庫時，總結整個專案的介紹。

![](https://pic.yupi.icu/1/image-20260109101803122.png)

Monica 最大的優勢是集成度高，幾乎所有常用的 AI 功能都有。而且它支援多個大模型，可以根據任務選擇最合適的模型，強烈安利。



### AITDK

[AITDK](https://aitdk.com/) 是 SEO 和內容創作神器，我用它來進行關鍵詞分析和競品研究。

做自媒體的朋友都知道，關鍵詞研究很重要。AITDK 可以幫我分析哪些關鍵詞搜索量大、競爭小，然後我就針對這些關鍵詞寫文章、或者優化自己的網站。

而且 AITDK 還能分析競品的內容策略，看看別人都在寫什麼，我就能找到差異化的角度。

![](https://pic.yupi.icu/1/image-20260109101947603.png)

## 九、我的使用策略

分享了這麼多工具，簡單總結一下：

- **做大專案時**，我會用 Cursor + Claude Opus 4.5，因為程式碼質量要求高，需要最強的 AI 能力。

- **做小專案或快速驗證想法時**，我會用 Claude Code + 智譜 GLM，因為速度快，成本低。

- **使用 VS Code 閱讀專案時**，我會搭配 Cline + DeepSeek 大模型，比較方便。

- **需要生成圖片時**，我會用 Nano Banana，因為中文支援好，生成速度快。

- **需要生成影片時**，我會根據質量要求選擇即夢 AI（性價比高）或 Veo 3（質量最高）。

- **日常遇到問題時**，我會直接問豆包或元寶，因為方便快捷。

- **寫文章需要配圖時**，我會用文心助手處理圖片，因為功能全面。

根據場景選擇最合適的工具，這樣既能保證效率，又能控制成本。
